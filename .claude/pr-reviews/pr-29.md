# PR Review: #29 — fix: V8 audit — XP bonuses, quest exIds, ymove removal, data quality

**Branch**: `claude/nervous-ellis` → `main`
**Reviewed**: 2026-04-05

## Summary

This PR addresses several data quality and correctness issues from a V8 audit: fixes incorrect quest `exId` values that tracked the wrong exercises, removes ymove references, adds missing `yoga_session` and `deep_stretch` exercises, adds `tricep` as a muscle group filter option, and extends `calcExXP` with `weightLbs` and `hrZone` bonus parameters. It also introduces a new `exerciseLibrary.js` utility for Supabase-backed exercise loading and video URL fetching, and fixes `checkQuestCompletion` to prefer `e.exId` over name-lookup.

---

## Issues Found

### 🔴 Critical

**1. `exerciseLibrary.js` uses `sb` without importing it**
- **File**: `src/utils/exerciseLibrary.js` (lines 22, 126, 147)
- The file calls `sb.from(...)` and `sb.rpc(...)` but never imports `sb` from `../utils/supabase`. This will throw `ReferenceError: sb is not defined` at runtime whenever `loadExercises()` or `getFreshVideoUrl()` is called.

**2. `calcPlanXP` and `calcDayXP` don't pass `weightLbs`/`hrZone` to updated `calcExXP`**
- **File**: `src/utils/xp.js` (lines 121, 125)
- The PR updates `calcExXP` to accept `weightLbs` and `hrZone`, but the helper wrappers `calcPlanXP` and `calcDayXP` still call it with only 5 arguments. Plans and day-view XP totals will always compute with `weightBonus=1` and `zoneBonus=1`, silently under-reporting XP.

**3. `q_cycle_10` quest still references non-existent `exId: "cycling"`**
- **File**: `src/data/constants.js` (line 196)
- The PR fixes yoga/stretch quest IDs but misses the cycling quest. `q_cycle_10` ("Road Rider") still has `auto:{exId:"cycling"}`, but the canonical exercise ID is `"cycle_ride"`. Cycling sessions will never complete this quest.

---

### 🟡 Moderate

**4. Several `calcExXP` call sites in App.js still omit `weightLbs`/`hrZone`**
- **File**: `src/App.js` (lines 2063, 4080–4085, 4355, 4466, 4584–4585, 4629, 4701, 8854, 9002)
- Multiple preview/display call sites pass only 5-6 arguments instead of 8, so the displayed XP previews won't reflect weight or HR zone bonuses even after the PR's main log path is fixed.

**5. `exerciseLibrary.js` mutates exported `EXERCISES` array and `EX_BY_ID` object directly**
- **File**: `src/utils/exerciseLibrary.js` (lines 71–80, 93–96)
- Directly mutating the module-level `EXERCISES` array and `EX_BY_ID` object is fragile — components that imported these before `loadExercises()` runs will silently have stale data. The `Object.assign(EX_BY_ID, ...)` rebuild also doesn't remove keys, meaning deleted exercises persist.

---

### 🟢 Minor

- `exerciseLibrary.js` imports `EX_BY_ID` from `../data/constants` but the file lives in `src/utils/` — the path should be correct, but worth confirming relative imports resolve.
- `yoga_session` and `deep_stretch` added to `EXERCISES` have no `xpClassMap` — they'll use the category-bonus fallback path which is fine, but inconsistent with other exercises that have explicit class maps.
- The "Ghost Run" plan template still uses `jumprope` in the PR diff but main-branch uses `jumpRope` (camelCase) — the PR doesn't change this, but the inconsistency was pre-existing.
