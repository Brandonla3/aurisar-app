## Fix Suggestions

### Issue #4 — Wrong muscleGroup on tricep-dominant exercises
**`src/data/exercises.js`** — Several new exercises have `muscleGroup:"bicep"` when they primarily target triceps.

**Why it matters**: MuscleGroup drives exercise filtering, icon coloring, and the muscle picker. Users filtering by "bicep" would see tricep exercises.

**Fix**: Change `muscleGroup` to `"bicep"` stays as the app convention (since the app maps triceps under "bicep" group — verify this is intentional in the existing data). If not, these should use the correct muscle group.

### Issue #5 — Reusable workout descriptions permanently collapsed with no toggle
**`src/App.js:3987`** — The `recipe-desc-collapsed` class is added but there's no click handler to expand.

**Why it matters**: Users who write long descriptions for their custom workouts will only ever see one truncated line.

**Fix**: Either add a toggle mechanism (like the recipe cards have) or use CSS `text-overflow:ellipsis` with a title attribute so users can hover to see the full text.

### Issue #6 — String reps cause NaN XP calculations
**`src/data/constants.js`** — Many templates use string reps like `"AMRAP"`, `"30s"`, `"10 each"`.

**Why it matters**: `calcExXP` likely does `parseInt(reps)` which returns `NaN` for non-numeric strings, resulting in `NaN` XP display in the recipe cards.

**Fix**: Either normalize reps to numbers in the template data (e.g., AMRAP→15, "10 each"→10, "30s"→30), or update `calcExXP` to handle string reps gracefully with a fallback like `parseInt(reps) || 1`.

### Issue #7 — !important on white-space breaks expanded descriptions
**`src/styles/app.css`** — `recipe-desc-collapsed` has `white-space:nowrap !important`.

**Why it matters**: When the class is removed (description expanded), the inline style `white-space:pre-line` should take effect, but if any parent class still applies, the `!important` could leak.

**Fix**: Remove `!important` — the class specificity is sufficient, or use a max-height approach instead of white-space for the collapse effect.

## Summary

This is a large data-driven PR that adds 50 workout recipes and 41 exercises from an external spreadsheet. The data mapping work is solid — exercise IDs were verified programmatically and superset pairings are correct. The main concerns are: (1) string reps like "AMRAP" and "30s" may cause NaN in XP calculations, (2) reusable workout descriptions are permanently collapsed with no expand toggle, and (3) a few muscleGroup misclassifications on tricep exercises. The UI improvements (filter bar, collapsible descriptions, superset rendering) are well-implemented and use existing design patterns.
