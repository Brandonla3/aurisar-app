# Aurisar — Workout Analytics & Trends Feature Summary
# =====================================================

## Overview
Added a Trends/Analytics section to the Battle Record tab, giving users interactive
charts and insights about their workout habits. Users can visualize which days of the
week they train best, track muscle group training frequency, monitor volume progression
per muscle group for hypertrophy, see sets-per-session trends, and more. All chart cards
are reorderable via drag-and-drop (desktop) and up/down arrow buttons (mobile).

## What Was Added

### New Dependency
- `recharts` (v3.8.1) — React-first charting library for rendering bar, line, and
  area charts with full dark-theme support

### New File: src/components/TrendsTab.js
Self-contained component containing all analytics logic:
- Time-range filtering (7 Days / 30 Days / 90 Days / 6 Months / All Time)
- 6 chart cards with memoized data aggregation for performance
- Chart reordering system (drag-and-drop + up/down arrows)
- Empty state handling when no workout data exists
- Dark glassmorphism styling matching the existing app theme
- Unit-aware display (imperial/metric) for volume calculations
- Exports DEFAULT_CHART_ORDER constant for App.js integration

## Chart Details

### 1. Best Training Days (Bar Chart) — key: "dow"
- 7 bars representing Sunday through Saturday
- Metric toggle: Sessions / Duration / Total Cal / Active Cal
- Best day highlighted in bright gold (#f0d060), others in standard gold (#c49428)
- Bar opacity scales proportionally to show relative intensity
- Auto-generated insight: "Your strongest day is [Day] with [X] [metric]"
- Deduplicates workout sessions by sourceGroupId to avoid double-counting

### 2. Sets Over Time (Bar Chart) — key: "sets"
- Visualizes average total sets per session each week
- Session definition: Solo exercise (no sourceGroupId) = its own session;
  entries sharing the same dateKey + sourceGroupId = one workout session
- Sums all sets across exercises in each session, then averages per week
- Gold bars (#c49428) with week labels on x-axis
- Insight: "Average total sets per session each week — ensure sufficient training stimulus"

### 3. Muscle Group Frequency (Horizontal Bar Chart) — key: "muscleFreq"
- Tracks how often specific muscles are trained per week
- Groups log entries by exercise muscleGroup field
- Counts distinct weeks each muscle group appears in
- Computes avgPerWeek = totalWeeksActive / totalDistinctWeeks in range
- Bars colored by MUSCLE_COLORS (chest=#9E7B6B, back=#7B8D7A, etc.)
- Auto-generated insight highlighting most trained muscle group

### 4. Volume Over Time (Stacked/Filtered Bar Chart) — key: "volume"
- Displays total load (sets × reps × weight) per muscle group each week
- Dropdown filter: "All Muscle Groups" (stacked bars) or specific muscle (single-color bars)
- Stacked bars colored by MUSCLE_COLORS per muscle group
- Supports metric/imperial unit conversion (lbs/kg)
- Purpose: Track muscle hypertrophy stimulus over time

### 5. Workout Consistency (Area Chart) — key: "consistency"
- Sessions logged per week as a filled area chart
- Fill color uses the user's character class color at low opacity
- Only appears when 2+ weeks of data exist
- X-axis labels show week start dates (e.g., "Mar 3")

### 6. Most Trained Exercises (Ranked List) — key: "topEx"
- Top 10 most-logged exercises sorted by frequency
- Each row shows: rank number, exercise icon, name, progress bar, count, and XP earned
- Progress bars colored by exercise category using CAT_ICON_COLORS
- Bar width proportional to the top exercise's count

## Chart Reordering System

### How It Works
- Each chart card has a header with the title + reorder controls (top-right)
- Controls: ↑ button (move up), ↓ button (move down), ⋮⋮ drag handle
- First chart's ↑ button is disabled; last chart's ↓ button is disabled
- Drag handle uses native HTML5 drag events (same pattern as existing plan/workout builders)
- On touch devices (@media hover:none), drag handle is hidden — arrows are the primary method

### Persistence
- Chart order stored as `profile.chartOrder` array (e.g., ["dow","sets","muscleFreq","volume","consistency","topEx"])
- Saved via existing doSave() pattern to both localStorage and Supabase
- Default order defined in DEFAULT_CHART_ORDER constant
- Falls back to default if saved order is missing or has wrong length

### Implementation
- Charts defined in CHART_RENDERERS map (key → render function)
- chartOrder array determines render sequence
- ChartCard wrapper component handles drag events and arrow button callbacks
- moveChart(fromIdx, toIdx) function splices the order array and calls onChartOrderChange prop
- App.js passes chartOrder from profile and onChartOrderChange callback that updates profile state

## Architecture Decisions

### Why Battle Record Sub-Tab (not a new main tab)?
- Battle Record is the app's data/history hub — trends are a natural extension
- Avoids crowding the existing 5-tab main navigation bar
- Leverages the existing logSubTab state and log-subtab-bar/log-subtab-btn CSS
- Zero new navigation patterns needed

### Why a Separate Component File?
- App.js is already 10,000+ lines with 252 useState hooks
- TrendsTab manages its own internal state (range, metric toggles, filters, drag state)
- Isolates charting library imports so recharts only loads when needed
- All data passed via props: log, allExById, clsColor, units, chartOrder, onChartOrderChange

### Why Native Drag + Arrow Buttons (no library)?
- App already uses native HTML5 drag in plan/workout builders (App.js lines 4691-4699, 5514-5519)
- Zero additional dependencies
- Arrow buttons work universally on mobile and desktop
- Drag handle hidden on touch devices to avoid UX issues
- ~30 lines of reorder logic total

### Why Recharts?
- React-first: works naturally with the existing React.createElement() pattern
- Responsive out of the box via ResponsiveContainer
- Easy to customize colors, tooltips, and styling for the dark theme
- Well-maintained with React 19 compatibility
- ~45KB gzipped — reasonable bundle size addition

## Data Flow
1. App.js passes `profile.log`, `allExById`, `cls.color`, `profile.units`, `profile.chartOrder`, and `onChartOrderChange` as props
2. TrendsTab filters log entries by selected time range (useMemo)
3. Each chart has its own useMemo hook that aggregates the filtered data
4. Workout sessions are deduplicated by sourceGroupId where appropriate
5. Chart order changes call onChartOrderChange which updates profile state (auto-saved)
6. No new state is added to App.js — all analytics state is local to TrendsTab

## Navigation Path
Main App -> Hamburger Menu -> "Battle Log" -> Sub-tabs -> "📊 Trends"
(activeTab === "history" -> logSubTab === "trends")
